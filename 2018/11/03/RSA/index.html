RSA非对称加密的简单理解

2018-11-03
chankin
Algorithm
RSA

RSA是一种公开密钥算法。通常是通讯双方各成一对RSA公秘钥,公钥和私钥都可以用于加密和解密。其中私钥用户保存(不公开),公钥对外公开，任何人都可以获取。

假设A给B发送文本 RSA签名加密过程如下：

  1. B传递自己的公钥给A;
  2. A对明文进行Hash运算得到信息摘要,用自己的私钥对摘要进行加密,形成数字签名, 同时使用B发送给A的公钥对明文加密,形成密文;
  3. 将密文和数字签名发送给B;
  4. B收到 A发过来的签名和密文, 获得A的公钥对数字签名进行解签, 进行Hash算法得到信息摘要;
  5. 再用自己的私钥对进行解密,得到明文,对明文进行Hash算法得到信息摘要;
  6. 将两个摘要进行对比,两者一致则说明文本没有被篡改过。

数字签名的作用：

证明消息发送源头,防止别人假冒和对消息进行篡改。

Hash算法：

数字摘要技术（Digital Digest）也称作为安全HASH编码法（SHA：Secure Hash Algorithm）。

它是一种摘要算法不是加密算法, 摘要算法是单向的, 明文可以通过摘要算法得到摘要结果。反之则不能通过摘要结果还原成明文，所以不可逆。

总结：公钥加密、私钥解密、私钥签名、公钥验签。
